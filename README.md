# ll-hls4ml

Surrogate models for predicting HLS resource usage and latency from LLVM Control/Data Flow Graphs (CDFGs).

## Data layout

CDFG JSON graphs live outside this repo (generated by `hls4ml_pipeline`):

```
../data/graphs/{kernel_type}/archive_{n}/*.json
../data/tensors/{kernel_type}/archive_{n}/*.pt   # PyG HeteroData (preprocessed)
```

Each JSON graph has `nodes`, `links`, and `labels` (LUT, FF, DSP, cycles, etc.).

## Install

```bash
pip install -e .
```

Override data location:

```bash
export LL_HLS4ML_DATA_ROOT=/path/to/data
```

## Workflow

1. **EDA** — `notebooks/01_eda_tabular.ipynb`: handcrafted graph features, PCA/t-SNE
2. **Preprocess** — `notebooks/02_preprocess_tensors.ipynb`: vocab scan, JSON → `.pt` tensors
3. **Train** — `notebooks/03_train_rgcn.ipynb`: heterogeneous R-GCN experiments

Batch scripts (optional):

```bash
python scripts/build_vocab.py
python scripts/build_tensors.py
python scripts/train.py
```

## Package structure

```
src/ll_hls4ml/
  io/          # CDFG schema, graph discovery, JSON loading
  data/        # vocab, tensorize, dataset, splits
  features/    # tabular feature extraction for EDA
  models/      # GNN architectures + registry
  training/    # loops, loaders, target normalization
  viz/         # dataset, EDA, and training plots
```

## Boundary with hls4ml_pipeline

| Repo | Role |
|------|------|
| `hls4ml_pipeline` | Download archives, extract LLVM IR, run ProGraML → write JSON to `../data/graphs` |
| `ll-hls4ml` | ML: exploration, preprocessing, training, evaluation |
